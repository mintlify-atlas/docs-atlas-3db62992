MMDetection organizes every neural network component — detectors, backbones, necks, heads, and losses — in a unified MMEngine registry. You can look up any class by its string name in a config file, or build models programmatically using `MODELS.build`. This page lists all registered classes and shows common instantiation patterns.

## Instantiating a model from config

<Tabs>
  <Tab title="From a config file">
    ```python
    from mmengine.config import Config
    from mmdet.registry import MODELS

    cfg = Config.fromfile('configs/rtmdet/rtmdet_s_8xb32-300e_coco.py')
    model = MODELS.build(cfg.model)
    ```
  </Tab>
  <Tab title="Using init_detector">
    ```python
    from mmdet.apis import init_detector

    model = init_detector(
        config='configs/faster_rcnn/faster-rcnn_r50_fpn_1x_coco.py',
        checkpoint='checkpoints/faster_rcnn_r50_fpn_1x_coco.pth',
        device='cuda:0',
    )
    model.eval()
    ```
  </Tab>
  <Tab title="Programmatic construction">
    ```python
    from mmdet.registry import MODELS

    model_cfg = dict(
        type='RTMDet',
        backbone=dict(type='CSPNeXt', arch='P5', out_indices=(2, 3, 4)),
        neck=dict(
            type='CSPNeXtPAFPN',
            in_channels=[128, 256, 512],
            out_channels=128,
        ),
        bbox_head=dict(
            type='RTMDetHead',
            num_classes=80,
            in_channels=128,
            stacked_convs=2,
        ),
        train_cfg=None,
        test_cfg=dict(score_thr=0.01, nms=dict(type='nms', iou_threshold=0.65)),
    )
    model = MODELS.build(model_cfg)
    ```
  </Tab>
</Tabs>

## Registry lookup

The `MODELS` registry lets you list or retrieve any registered class:

```python
from mmdet.registry import MODELS

# Check if a class is registered
print('RTMDet' in MODELS)  # True

# Retrieve the class object
RTMDet = MODELS.get('RTMDet')

# List all registered detector names
names = list(MODELS._module_dict.keys())
```

<Note>
Custom models must call `@MODELS.register_module()` before they can be referenced by name in a config. See the [Customize Models](/advanced-guides/customize-models) guide for the full workflow.
</Note>
