MMDetection ships a collection of analysis and utility scripts under `tools/`. These cover inference speed benchmarking, training curve analysis, dataset browsing, model complexity measurement, and more.

---

## Log analysis

`tools/analysis_tools/analyze_logs.py` plots loss and mAP curves from a training log file, and can also compute average training speed per iteration.

<Note>
Install the plotting dependency before use: `pip install seaborn`
</Note>

```bash
python tools/analysis_tools/analyze_logs.py plot_curve \
    [--keys ${KEYS}] \
    [--eval-interval ${EVALUATION_INTERVAL}] \
    [--title ${TITLE}] \
    [--legend ${LEGEND}] \
    [--backend ${BACKEND}] \
    [--style ${STYLE}] \
    [--out ${OUT_FILE}]
```

**Examples:**

```bash
# Plot classification loss
python tools/analysis_tools/analyze_logs.py plot_curve \
    log.json --keys loss_cls --legend loss_cls

# Plot classification and regression loss, save as PDF
python tools/analysis_tools/analyze_logs.py plot_curve \
    log.json --keys loss_cls loss_bbox --out losses.pdf

# Compare bbox mAP across two runs
python tools/analysis_tools/analyze_logs.py plot_curve \
    log1.json log2.json --keys bbox_mAP --legend run1 run2

# Compute average training speed
python tools/analysis_tools/analyze_logs.py cal_train_time \
    log.json [--include-outliers]
```

## FPS benchmark

`tools/analysis_tools/benchmark.py` measures model inference speed in frames per second (FPS). The reported value includes both forward pass and post-processing. It uses single-GPU distributed launch mode for accuracy:

```bash
python -m torch.distributed.launch \
    --nproc_per_node=1 \
    --master_port=${PORT} \
    tools/analysis_tools/benchmark.py \
    ${CONFIG} \
    [--checkpoint ${CHECKPOINT}] \
    [--repeat-num ${REPEAT_NUM}] \
    [--max-iter ${MAX_ITER}] \
    [--log-interval ${LOG_INTERVAL}] \
    --launcher pytorch
```

**Example:**

```bash
python -m torch.distributed.launch \
    --nproc_per_node=1 \
    --master_port=29500 \
    tools/analysis_tools/benchmark.py \
    configs/faster_rcnn/faster-rcnn_r50_fpn_1x_coco.py \
    checkpoints/faster_rcnn_r50_fpn_1x_coco_20200130-047c8118.pth \
    --launcher pytorch
```

## Dataset browsing

`tools/analysis_tools/browse_dataset.py` visualizes dataset images with ground-truth bounding box annotations after data augmentation transforms are applied. Use it to verify your pipeline configuration is correct before training.

```bash
python tools/analysis_tools/browse_dataset.py \
    ${CONFIG} \
    [--skip-type ${SKIP_TYPE[SKIP_TYPE...]}] \
    [--output-dir ${OUTPUT_DIR}] \
    [--not-show] \
    [--show-interval ${SHOW_INTERVAL}]
```

## Model complexity (FLOPs and parameters)

`tools/analysis_tools/get_flops.py` computes FLOPs and parameter count for a given model configuration:

```bash
python tools/analysis_tools/get_flops.py \
    ${CONFIG_FILE} \
    [--shape ${INPUT_SHAPE}]
```

**Example output:**

```text
==============================
Input shape: (3, 1280, 800)
Flops: 239.32 GFLOPs
Params: 37.74 M
==============================
```

<Warning>
FLOPs depend on the input shape (default: `1, 3, 1280, 800`). Some custom operators such as GN are not counted. Treat results as estimates for comparison, not authoritative numbers for papers.
</Warning>

## Result analysis

`tools/analysis_tools/analyze_results.py` computes per-image mAP and saves the images with the highest and lowest detection scores, which is useful for diagnosing failure cases.

```bash
python tools/analysis_tools/analyze_results.py \
    ${CONFIG} \
    ${PREDICTION_PATH} \
    ${SHOW_DIR} \
    [--show] \
    [--topk ${TOPK}] \
    [--show-score-thr ${SHOW_SCORE_THR}] \
    [--cfg-options ${CFG_OPTIONS}]
```

| Argument | Description |
|---|---|
| `prediction_path` | Pickle result file from `tools/test.py --out`. |
| `show_dir` | Directory to save annotated images. |
| `--topk` | Number of best and worst images to save (default: 20). |
| `--show-score-thr` | Minimum score for displayed detections (default: 0). |

**Example:**

```bash
python tools/analysis_tools/analyze_results.py \
    configs/faster_rcnn/faster-rcnn_r50_fpn_1x_coco.py \
    result.pkl \
    results/ \
    --topk 50 \
    --show-score-thr 0.3
```

## Robustness testing

`tools/analysis_tools/test_robustness.py` and `tools/analysis_tools/robustness_eval.py` evaluate model robustness on corrupted images. The methodology follows [Benchmarking Robustness in Object Detection](https://arxiv.org/abs/1907.07484).

Refer to `docs/en/user_guides/robustness_benchmarking.md` for the full workflow including which corruption types are tested and how to interpret the results.

## Confusion matrix

`tools/analysis_tools/confusion_matrix.py` analyzes prediction results and plots a per-class confusion matrix.

<Steps>
  <Step title="Generate prediction results">
    ```bash
    python tools/test.py ${CONFIG} ${MODEL_PATH} --out results.pkl
    ```
  </Step>
  <Step title="Plot the confusion matrix">
    ```bash
    python tools/analysis_tools/confusion_matrix.py \
        ${CONFIG} \
        results.pkl \
        ${SAVE_DIR} \
        --show
    ```
  </Step>
</Steps>

## Fusing predictions from multiple models

`tools/analysis_tools/fuse_results.py` merges predictions from multiple models using Weighted Boxes Fusion (WBF):

```bash
python tools/analysis_tools/fuse_results.py \
    ${PRED_RESULT_1} ${PRED_RESULT_2} ${PRED_RESULT_3} \
    [--annotation ${ANNOTATION}] \
    [--weights ${WEIGHTS}] \
    [--fusion-iou-thr ${FUSION_IOU_THR}] \
    [--save-fusion-results] \
    [--out-dir ${OUT_DIR}]
```

**Example — fuse three models and evaluate:**

```bash
python tools/analysis_tools/fuse_results.py \
    faster-rcnn_r50-caffe_fpn_1x_coco.json \
    retinanet_r50-caffe_fpn_1x_coco.json \
    cascade-rcnn_r50-caffe_fpn_1x_coco.json \
    --annotation annotation.json \
    --weights 1 2 3
```

<Note>
WBF fusion currently supports COCO format result files only.
</Note>

## Dataset download

`tools/misc/download_dataset.py` supports downloading COCO, VOC, and LVIS directly:

```bash
python tools/misc/download_dataset.py --dataset-name coco2017
python tools/misc/download_dataset.py --dataset-name voc2007
python tools/misc/download_dataset.py --dataset-name lvis
```

To inspect the complete, fully-resolved config including all inherited values:

```bash
python tools/misc/print_config.py ${CONFIG} [--options ${OPTIONS}]
```

## YOLO anchor optimization

`tools/analysis_tools/optimize_anchors.py` provides two methods to fit YOLO anchor sizes to your dataset:

<Tabs>
  <Tab title="K-means clustering">
    ```bash
    python tools/analysis_tools/optimize_anchors.py \
        ${CONFIG} \
        --algorithm k-means \
        --input-shape ${WIDTH} ${HEIGHT} \
        --output-dir ${OUTPUT_DIR}
    ```
  </Tab>
  <Tab title="Differential evolution">
    ```bash
    python tools/analysis_tools/optimize_anchors.py \
        ${CONFIG} \
        --algorithm differential_evolution \
        --input-shape ${WIDTH} ${HEIGHT} \
        --device cuda \
        --output-dir ${OUTPUT_DIR}
    ```
  </Tab>
</Tabs>
