MMDetection provides `tools/train.py` and `tools/dist_train.sh` for training detection models on standard and custom datasets. All training modes share the same config-driven workflow — specify a config file and optionally override settings on the command line.

<Note>
Before training, make sure your dataset is prepared under `data/`. See the [dataset preparation guide](/user-guides/dataset-preparation) for details.
</Note>

---

## Single GPU training

```bash
python tools/train.py \
    ${CONFIG_FILE} \
    [optional arguments]
```

**Example:**

```bash
python tools/train.py \
    configs/faster_rcnn/faster-rcnn_r50_fpn_1x_coco.py \
    --work-dir work_dirs/faster_rcnn_r50
```

Logs and checkpoints are saved to `work_dir` (defaults to `./work_dirs/{config_name}` if not set).

### CPU-only training

```bash
export CUDA_VISIBLE_DEVICES=-1
python tools/train.py ${CONFIG_FILE}
```

<Warning>
CPU training is extremely slow. Use it only for debugging on machines without GPUs.
</Warning>

## Multi-GPU distributed training

```bash
bash tools/dist_train.sh \
    ${CONFIG_FILE} \
    ${GPU_NUM} \
    [optional arguments]
```

**Example — 8 GPUs:**

```bash
bash tools/dist_train.sh \
    configs/mask_rcnn/mask-rcnn_r50_fpn_1x_coco.py \
    8 \
    --work-dir work_dirs/mask_rcnn_r50_8gpu
```

### Running two jobs on one machine

When launching multiple jobs on the same machine, specify different ports to avoid communication conflicts:

```bash
CUDA_VISIBLE_DEVICES=0,1,2,3 PORT=29500 \
    bash tools/dist_train.sh ${CONFIG_FILE} 4

CUDA_VISIBLE_DEVICES=4,5,6,7 PORT=29501 \
    bash tools/dist_train.sh ${CONFIG_FILE} 4
```

### Multi-machine training

<Tabs>
  <Tab title="Machine 0 (master)">
    ```bash
    NNODES=2 NODE_RANK=0 \
        PORT=$MASTER_PORT \
        MASTER_ADDR=$MASTER_ADDR \
        bash tools/dist_train.sh $CONFIG $GPUS
    ```
  </Tab>
  <Tab title="Machine 1">
    ```bash
    NNODES=2 NODE_RANK=1 \
        PORT=$MASTER_PORT \
        MASTER_ADDR=$MASTER_ADDR \
        bash tools/dist_train.sh $CONFIG $GPUS
    ```
  </Tab>
</Tabs>

<Note>
Multi-machine training without high-speed networking (e.g. InfiniBand) will be slow.
</Note>

## Slurm cluster training

Use `tools/slurm_train.sh` on clusters managed by [Slurm](https://slurm.schedmd.com/):

```bash
[GPUS=${GPUS}] bash tools/slurm_train.sh \
    ${PARTITION} \
    ${JOB_NAME} \
    ${CONFIG_FILE} \
    ${WORK_DIR}
```

**Example — 16 GPUs on the `dev` partition:**

```bash
GPUS=16 bash tools/slurm_train.sh \
    dev \
    mask_r50_1x \
    configs/mask_rcnn/mask-rcnn_r50_fpn_1x_coco.py \
    /nfs/shared/mask_rcnn_r50_fpn_1x
```

When running multiple Slurm jobs simultaneously, set unique ports for each:

```bash
# Job 1
CUDA_VISIBLE_DEVICES=0,1,2,3 GPUS=4 \
    bash tools/slurm_train.sh ${PARTITION} job1 config1.py ${WORK_DIR} \
    --cfg-options dist_params.port=29500

# Job 2
CUDA_VISIBLE_DEVICES=4,5,6,7 GPUS=4 \
    bash tools/slurm_train.sh ${PARTITION} job2 config2.py ${WORK_DIR} \
    --cfg-options dist_params.port=29501
```

## Key training flags

| Flag | Description |
|---|---|
| `--work-dir` | Directory to save logs and checkpoints. |
| `--resume` | Resume from the latest checkpoint in `work_dir` automatically. |
| `--resume ${CKPT}` | Resume from a specific checkpoint file. |
| `--cfg-options` | Override config values inline (e.g. `model.backbone.depth=101`). |
| `--amp` | Enable automatic mixed precision (AMP) training. |
| `--auto-scale-lr` | Automatically scale the learning rate based on actual batch size. |

### `resume` vs `load-from`

- `--resume` restores both model weights **and** optimizer state, continuing from the saved iteration number.
- `load_from` (set in config) loads model weights only; training restarts from epoch 0. Use it for fine-tuning.

## Mixed precision (AMP) training

Pass `--amp` to switch the optimizer wrapper to `AmpOptimWrapper` with dynamic loss scaling:

```bash
python tools/train.py \
    configs/faster_rcnn/faster-rcnn_r50_fpn_1x_coco.py \
    --amp
```

You can also set it directly in the config:

```python
optim_wrapper = dict(
    type='AmpOptimWrapper',
    optimizer=dict(type='SGD', lr=0.02, momentum=0.9, weight_decay=0.0001),
    loss_scale='dynamic')
```

## Automatic learning rate scaling

The default learning rate in MMDetection configs targets a batch size of 16 (8 GPUs × 2 images). If your setup differs, enable auto-scaling:

```bash
python tools/train.py \
    ${CONFIG_FILE} \
    --auto-scale-lr
```

This applies the [linear scaling rule](https://arxiv.org/abs/1706.02677) automatically. For example, scaling from 16 to 64 total images per step multiplies the base LR by 4.

<Warning>
Check that your config contains `auto_scale_lr.base_batch_size`. Configs with non-default batch sizes include this field at the bottom of the file.
</Warning>

## Testing and evaluation after training

Use `tools/test.py` to evaluate a trained checkpoint:

<Tabs>
  <Tab title="Single GPU">
    ```bash
    python tools/test.py \
        ${CONFIG_FILE} \
        ${CHECKPOINT_FILE} \
        [--show] \
        [--show-dir ${OUTPUT_DIR}]
    ```
  </Tab>
  <Tab title="Multi-GPU">
    ```bash
    bash tools/dist_test.sh \
        ${CONFIG_FILE} \
        ${CHECKPOINT_FILE} \
        ${GPU_NUM} \
        [--out results.pkl]
    ```
  </Tab>
</Tabs>

**Example:**

```bash
python tools/test.py \
    configs/faster_rcnn/faster-rcnn_r50_fpn_1x_coco.py \
    work_dirs/faster_rcnn_r50/epoch_12.pth \
    --show-dir results/
```

See the [inference guide](/user-guides/inference) for more details on running predictions.
