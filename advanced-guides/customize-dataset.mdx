MMDetection's dataset system is built on top of MMEngine's `BaseDataset`. Every dataset class registers itself with the `DATASETS` registry so that the config system can construct it by name. This guide covers how to adapt new data in COCO JSON format, how to write a fully custom dataset class, and how to use dataset wrappers.

## Option 1: Use COCO JSON format (recommended)

Converting your annotations to COCO format is the simplest path. Once converted, you can use the built-in `CocoDataset` class and only adjust the config.

### COCO JSON structure

The annotation file must contain three top-level keys:

```json annotation.json
{
  "images": [
    {
      "file_name": "000001.jpg",
      "height": 720,
      "width": 1280,
      "id": 1
    }
  ],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "category_id": 1,
      "bbox": [10, 20, 40, 60],
      "area": 2400,
      "iscrowd": 0
    }
  ],
  "categories": [
    {"id": 1, "name": "cat"},
    {"id": 2, "name": "dog"}
  ]
}
```

<Warning>
For instance segmentation, `segmentation` entries must be polygon coordinates with at least three points. MMDetection currently evaluates mask AP only for datasets in COCO format.
</Warning>

### Config for a custom COCO-format dataset

```python configs/my_custom_config.py
_base_ = './cascade_mask_rcnn_r50_fpn_1x_coco.py'

dataset_type = 'CocoDataset'
classes = ('a', 'b', 'c', 'd', 'e')
data_root = 'path/to/your/'

train_dataloader = dict(
    batch_size=2,
    dataset=dict(
        type=dataset_type,
        metainfo=dict(classes=classes),
        data_root=data_root,
        ann_file='train/annotation.json',
        data_prefix=dict(img='train/images/')))

val_dataloader = dict(
    batch_size=1,
    dataset=dict(
        type=dataset_type,
        test_mode=True,
        metainfo=dict(classes=classes),
        data_root=data_root,
        ann_file='val/annotation.json',
        data_prefix=dict(img='val/images/')))

test_dataloader = dict(
    batch_size=1,
    dataset=dict(
        type=dataset_type,
        test_mode=True,
        metainfo=dict(classes=classes),
        data_root=data_root,
        ann_file='test/annotation.json',
        data_prefix=dict(img='test/images/')))

# Override num_classes in the model head
model = dict(
    roi_head=dict(
        bbox_head=[
            dict(type='Shared2FCBBoxHead', num_classes=5),
            dict(type='Shared2FCBBoxHead', num_classes=5),
            dict(type='Shared2FCBBoxHead', num_classes=5)],
        mask_head=dict(num_classes=5)))
```

<Note>
`classes` must be a tuple, not a list. MMDetection automatically maps non-contiguous `category_id` values in the JSON to contiguous label indices. The order of names in `classes` must match the order of names in the `categories` field of the annotation file.
</Note>

### Annotation validation checklist

Before training, verify:

1. The length of `categories` equals the number of entries in your `classes` tuple.
2. Every `category_id` in `annotations` appears in `categories`.
3. `name` values in `categories` are in the same order as your `classes` tuple.

## Option 2: Use MMEngine's middle format

If converting to COCO JSON is not practical, you can write an annotation file in MMEngine's native format directly and use `BaseDetDataset` without subclassing.

The annotation file (JSON, YAML, or pickle) must contain two fields:

```json annotation_middle.json
{
  "metainfo": {
    "classes": ["person", "bicycle", "car", "motorcycle"]
  },
  "data_list": [
    {
      "img_path": "images/000001.jpg",
      "height": 604,
      "width": 640,
      "instances": [
        {"bbox": [0, 0, 10, 20], "bbox_label": 0, "ignore_flag": 0},
        {"bbox": [10, 10, 110, 120], "bbox_label": 2, "ignore_flag": 0}
      ]
    }
  ]
}
```

Set `ignore_flag=1` for crowd or difficult instances that should be excluded from evaluation.

## Option 3: Write a fully custom dataset class

Use this approach when your annotation format cannot easily be converted offline.

<Steps>
  <Step title="Subclass BaseDetDataset">
    Create `mmdet/datasets/my_dataset.py`. Define `METAINFO` with class names and optionally a colour palette. Override `load_data_list` to parse your annotation file and return a list of dicts.

    The example below reads a plain-text annotation file where each image block is separated by a `#` line:

    ```python mmdet/datasets/my_dataset.py
    import mmengine

    from mmdet.datasets.base_det_dataset import BaseDetDataset
    from mmdet.registry import DATASETS


    @DATASETS.register_module()
    class MyDataset(BaseDetDataset):

        METAINFO = {
            'classes': ('person', 'bicycle', 'car', 'motorcycle'),
            'palette': [(220, 20, 60), (119, 11, 32), (0, 0, 142), (0, 0, 230)],
        }

        def load_data_list(self, ann_file):
            ann_list = mmengine.list_from_file(ann_file)
            data_infos = []

            for i, ann_line in enumerate(ann_list):
                if ann_line != '#':
                    continue

                img_path = ann_list[i + 1]
                width, height = ann_list[i + 2].split(' ')
                bbox_number = int(ann_list[i + 3])

                instances = []
                for ann in ann_list[i + 4 : i + 4 + bbox_number]:
                    parts = ann.split(' ')
                    instances.append({
                        'bbox': [float(v) for v in parts[:4]],
                        'bbox_label': int(parts[4]),
                        'ignore_flag': 0,
                    })

                data_infos.append(dict(
                    img_path=img_path,
                    width=int(width),
                    height=int(height),
                    instances=instances,
                ))

            return data_infos
    ```
  </Step>

  <Step title="Register the dataset">
    Add an import to `mmdet/datasets/__init__.py`:

    ```python mmdet/datasets/__init__.py
    from .my_dataset import MyDataset
    ```

    Or use `custom_imports` in your config:

    ```python configs/my_config.py
    custom_imports = dict(
        imports=['mmdet.datasets.my_dataset'],
        allow_failed_imports=False)
    ```
  </Step>

  <Step title="Use the dataset in a config">
    ```python configs/my_config.py
    train_dataloader = dict(
        dataset=dict(
            type='MyDataset',
            ann_file='path/to/annotation.txt',
            data_prefix=dict(img='path/to/images/'),
            pipeline=train_pipeline))
    ```
  </Step>
</Steps>

## Use dataset wrappers

MMEngine provides three wrappers for combining or rebalancing datasets. Configure them in the `train_dataloader` block:

<Tabs>
  <Tab title="RepeatDataset">
    Repeats the entire dataset a fixed number of times per epoch. Useful when your dataset is small relative to the training schedule.

    ```python configs/my_config.py
    train_dataloader = dict(
        dataset=dict(
            type='RepeatDataset',
            times=3,
            dataset=dict(
                type='MyDataset',
                ann_file='annotation.txt',
                pipeline=train_pipeline)))
    ```
  </Tab>
  <Tab title="ConcatDataset">
    Concatenates multiple datasets, treating them as one. Each sub-dataset can have a different type.

    ```python configs/my_config.py
    train_dataloader = dict(
        dataset=dict(
            type='ConcatDataset',
            datasets=[
                dict(type='CocoDataset', ann_file='coco_train.json', ...),
                dict(type='MyDataset', ann_file='my_train.txt', ...),
            ]))
    ```
  </Tab>
  <Tab title="ClassBalancedDataset">
    Repeats images in a class-balanced manner to address long-tail distributions.

    ```python configs/my_config.py
    train_dataloader = dict(
        dataset=dict(
            type='ClassBalancedDataset',
            oversample_thr=1e-3,
            dataset=dict(
                type='CocoDataset',
                ann_file='train.json',
                pipeline=train_pipeline)))
    ```
  </Tab>
</Tabs>

## Filter classes from an existing dataset

You can train on a subset of categories without writing a custom dataset class. Setting `metainfo.classes` causes the dataset to automatically ignore ground-truth boxes from other categories:

```python configs/three_class_config.py
classes = ('person', 'bicycle', 'car')

train_dataloader = dict(
    dataset=dict(metainfo=dict(classes=classes)))

val_dataloader = dict(
    dataset=dict(metainfo=dict(classes=classes)))

test_dataloader = dict(
    dataset=dict(metainfo=dict(classes=classes)))
```

<Warning>
When you restrict `classes`, update `num_classes` in the model head to match. The built-in `NumClassCheckHook` will raise an error if they are inconsistent.
</Warning>
