Understanding how data moves through MMDetection makes it far easier to write custom components and debug issues. Data passes through three major stages: the dataset and transform pipeline produces a `results` dict, the model converts that dict into a `DetDataSample`, and the model either computes losses (training) or produces `pred_instances` (inference). This page traces each transition in detail.

## Stage 1: Dataset loading and transform pipeline

The `results` dict originates in the dataset's `load_data_list` method (or in the COCO annotation parser). It starts as a minimal dict and grows as transforms add processed data.

### Initial dict from the dataset

```python example
# After load_data_list returns one item
results = {
    'img_path': 'images/000001.jpg',
    'img_id': 1,
    'height': 720,
    'width': 1280,
    'instances': [
        {'bbox': [10.0, 20.0, 50.0, 80.0], 'bbox_label': 0, 'ignore_flag': 0},
        {'bbox': [100.0, 150.0, 200.0, 300.0], 'bbox_label': 2, 'ignore_flag': 0},
    ],
}
```

### After LoadImageFromFile

`LoadImageFromFile` reads the image from disk and adds it to the dict:

```python example
results['img']       # np.ndarray, shape (H, W, 3), dtype uint8, BGR
results['img_shape'] # (720, 1280)
results['ori_shape'] # (720, 1280)
```

### After LoadAnnotations

`LoadAnnotations` converts the `instances` list into typed arrays:

```python example
results['gt_bboxes']       # HorizontalBoxes, shape (N, 4), xyxy format
results['gt_labels']       # np.ndarray, shape (N,), dtype int64
results['gt_ignore_flags'] # np.ndarray, shape (N,), dtype bool
# If with_mask=True:
results['gt_masks']        # PolygonMasks or BitmapMasks
```

### After Resize and spatial transforms

Each spatial transform updates `img`, `img_shape`, and adjusts `gt_bboxes` and `gt_masks` to match:

```python example
results['img']         # np.ndarray, resized, shape (H', W', 3)
results['img_shape']   # (H', W')
results['scale_factor'] # np.ndarray([w_scale, h_scale])
results['gt_bboxes']   # HorizontalBoxes scaled to new image size
```

### After PackDetInputs

`PackDetInputs` is the final transform. It converts the `results` dict into a `DetDataSample` and wraps the image tensor:

```python example
# Output structure consumed by the model's data_preprocessor
{
    'inputs': torch.Tensor,   # shape (3, H', W'), normalised
    'data_samples': DetDataSample,
}
```

## Stage 2: DetDataSample — the central data container

`DetDataSample` is the structured container that carries both ground truth and predictions. It is defined in `mmdet/structures/det_data_sample.py` and is passed between model components.

### Fields overview

| Field | Type | When populated | Description |
|-------|------|----------------|-------------|
| `gt_instances` | `InstanceData` | Training | Ground-truth boxes, labels, masks |
| `pred_instances` | `InstanceData` | Inference | Predicted boxes, scores, labels |
| `ignored_instances` | `InstanceData` | Training | Instances excluded from loss |
| `proposals` | `InstanceData` | Two-stage, training | Region proposals from RPN |
| `gt_panoptic_seg` | `PixelData` | Panoptic training | Panoptic ground-truth map |
| `pred_panoptic_seg` | `PixelData` | Panoptic inference | Predicted panoptic map |
| `gt_sem_seg` | `PixelData` | Semantic training | Semantic ground-truth map |
| `pred_sem_seg` | `PixelData` | Semantic inference | Predicted semantic map |
| `metainfo` | `dict` | Always | `img_shape`, `ori_shape`, `img_id`, etc. |

### InstanceData fields

`InstanceData` is a flexible container from MMEngine. Its fields are set dynamically and all share the same first dimension (number of instances):

```python example
# During training — gt_instances
data_sample.gt_instances.bboxes  # HorizontalBoxes (N, 4)
data_sample.gt_instances.labels  # torch.Tensor (N,)
data_sample.gt_instances.masks   # BitmapMasks or PolygonMasks

# During inference — pred_instances
data_sample.pred_instances.bboxes  # torch.Tensor (M, 4), xyxy in original image coords
data_sample.pred_instances.scores  # torch.Tensor (M,)
data_sample.pred_instances.labels  # torch.Tensor (M,)
```

### Creating a DetDataSample (example)

```python example
import torch
from mmengine.structures import InstanceData
from mmdet.structures import DetDataSample

data_sample = DetDataSample()
data_sample.set_metainfo({'img_shape': (800, 1196), 'ori_shape': (720, 1280)})

gt_instances = InstanceData()
gt_instances.bboxes = torch.tensor([[10., 20., 50., 80.]])
gt_instances.labels = torch.tensor([0])
data_sample.gt_instances = gt_instances
```

## Stage 3: Training data flow

During training, the model calls `loss()` rather than `predict()`. Here is how data moves inside a typical single-stage detector (e.g. RetinaNet):

<Steps>
  <Step title="Data preprocessor">
    `DetDataPreprocessor` stacks images into a batch tensor and moves `DetDataSample` objects to the GPU. It also pads images to the same shape within the batch.
  </Step>

  <Step title="Backbone">
    The image batch `(B, 3, H, W)` passes through the backbone and produces a tuple of multi-scale feature maps:

    ```python
    features = backbone(img)
    # features: (feat_s3, feat_s4, feat_s5)  — one tensor per stage
    ```
  </Step>

  <Step title="Neck">
    The feature tuple goes into the neck (e.g. FPN), which aggregates and returns another tuple of feature maps at the desired output scales:

    ```python
    neck_features = neck(features)
    # neck_features: (p3, p4, p5, p6, p7)
    ```
  </Step>

  <Step title="Head — loss computation">
    The head receives the neck features and the list of `DetDataSample` objects. It assigns ground-truth labels to anchors/points, computes `cls_score` and `bbox_pred` tensors, then calls the configured loss functions:

    ```python
    losses = bbox_head.loss(neck_features, batch_data_samples)
    # losses: {'loss_cls': tensor, 'loss_bbox': tensor}
    ```

    The returned dict is aggregated by `Runner` to produce the total loss scalar for backpropagation.
  </Step>
</Steps>

## Stage 4: Inference data flow

During inference, the model calls `predict()`. The head produces raw class scores and box deltas, then `postprocess` applies NMS and rescales boxes back to the original image dimensions.

<Steps>
  <Step title="Forward through backbone and neck">
    Same as training — image batch → backbone → neck → multi-scale features.
  </Step>

  <Step title="Head — decode predictions">
    The head's `predict` method converts raw output tensors into `InstanceData` and stores them on the `DetDataSample`:

    ```python
    result_list = bbox_head.predict(neck_features, batch_data_samples)
    # Each DetDataSample now has pred_instances with:
    #   .bboxes  — torch.Tensor (M, 4) in original image coordinates
    #   .scores  — torch.Tensor (M,)
    #   .labels  — torch.Tensor (M,)
    ```
  </Step>

  <Step title="NMS and score thresholding">
    The head applies NMS (configured in `test_cfg`) to remove duplicate detections. Boxes are rescaled from `img_shape` back to `ori_shape` using the stored `scale_factor`.
  </Step>

  <Step title="Output">
    The final list of `DetDataSample` objects is returned by the model's `forward`. Each item's `pred_instances` contains the final detections for one image.
  </Step>
</Steps>

## Two-stage detector data flow

Two-stage detectors (Faster R-CNN, Cascade R-CNN) have an additional `proposals` step between the neck and the final head:

```python example — two-stage forward
# Stage 1: RPN
rpn_features = neck(backbone(img))
proposals = rpn_head.predict(rpn_features, batch_data_samples)
# proposals stored as data_sample.proposals (InstanceData)

# Stage 2: RoI head
roi_features = roi_extractor(rpn_features, proposals)
results = bbox_head(roi_features)
# cls_score, bbox_pred → decode → pred_instances
```

During training the RPN and RoI head each produce their own loss dicts, which are merged and returned together.

## Data flow summary

```
disk
  └─ load_data_list()   → results dict {img_path, instances, ...}
       └─ LoadImageFromFile  → adds results['img']
            └─ LoadAnnotations → adds results['gt_bboxes'], gt_labels, ...
                 └─ Resize, RandomFlip, ...  → transforms img + bboxes
                      └─ PackDetInputs  → DetDataSample (gt_instances)
                           └─ DataPreprocessor  → GPU batch
                                └─ Backbone → feature maps
                                     └─ Neck → multi-scale features
                                          ├─ Training: loss() → dict of loss scalars
                                          └─ Inference: predict() → pred_instances (bboxes, scores, labels)
```

<Tip>
To inspect the `results` dict at any point in the pipeline, add a print statement inside a temporary transform and insert it at the desired position. The `browse_dataset.py` tool also visualises the dict state after the full pipeline runs.
</Tip>
